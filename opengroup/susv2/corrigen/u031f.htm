Change Number:  U031/XNS-001

Title:          Incorrect specification of "type" for getnetbyaddr()

Qualifier:      Major technical

Rationale:      The description in the endnetent() reference
                page (Page 190) of the "net" argument of getnetbyaddr() 
                being of type "in_addr_t" is an erratum. It should 
                be of type "uint32_t", as in the getnetbyaddr() reference
                page (Page 196) and on Page 212.

                [Page numbers refer to The Open Group's printed
                XNS, Issue 5 publication.]

Change:         Change the endnetent() reference page from:
                
                  struct netent *getnetbyaddr(in_addr_t net, int type);

                to:

                  struct netent *getnetbyaddr(uint32_t net, int type);

Change Number:  U031/XNS-002

Title:          Typographical error

Qualifier:      Minor editorial

Rationale:      In the list of thread-safe interfaces in Section 1.5, 
                Thread Safety, the first item in the list is 
                incompletely shown.

Change:         Change the first item in the list of thread-safe
                interfaces shown in Section 1.5, Thread Safety from:
                
                  byaddr()
